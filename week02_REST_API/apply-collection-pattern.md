# 학습 키워드
## CQS(Command Query Separation)
- CQS는 Command와 Query를 분리한 개념으로 각각 각각의 행위의 결과가 비슷한것끼리 모아 분리한것이다.

    - Command : command는 데이터의 상태를 변화(Update)시키거나 생성(Create) 또는 삭제(Delete)하는 작업을 하지만 값을 반환하지 않는다.

    - Query : query는 데이터를 읽기(Read)만 하는 작업을 한다. 그래서 command와 다르게 읽은 데이터를 반환할 수 있다.
        - Query는 읽기만 하는 작업이기 떄문에 성능을 높이기 위해 캐시 전략을 고려할 수도 있다.

## CQS를 적용함으로써 얻을 수 있는 이점
- 코드 가독성 향상 : 아무래도 각각 관심사에 맞게 분리를 하기 때문에 어떤 코드가 데이터를 읽기만 하는지 아니면 데이터를 생성하거나 삭제하는지 단번에 파악하기가 쉬워진다.

- 코드 유지보수성 향상 : command와 query를 분리함으로써 각각 독립적이기 떄문에 이후 유지보수하기가 쉽다. 즉 command를 변경해도 query에 영향을 미치지 않고, query를 변경해도 command에 영향을 미치지 않기 때문에 유지보수나 추후 확장하기도 좋다.

- 디버깅 용이 : 에러가 발생했을때, 어떤 작업을 하는 도중에 에러가 발생했는지 한번에 파악하기가 쉬워지고 덕분에 디버깅하는 시간을 줄일 수 있다.
